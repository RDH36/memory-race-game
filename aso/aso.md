# ASO Google Play — Flipia

> Stratégie ASO complète pour **Flipia**, jeu de mémoire compétitif (cartes à apparier) avec mode multijoueur en ligne (ELO), IA, et power-up "Tornade".

**Package** : `com.rdh36.flipia`
**Catégorie principale** : Jeux > Cartes (ou Jeux > Réflexion / Casse-tête)
**Catégorie secondaire** : Jeux > Casual
**Marchés prioritaires** : 🇫🇷 France / Francophonie · 🇲🇬 Madagascar · 🌍 International (EN)

---

## 🧭 Positionnement & Audience

### Proposition de valeur
> **Le jeu de mémoire qui devient un sport.** Affronte de vrais joueurs en ligne, grimpe au classement ELO, et utilise la **Tornade** pour effacer la mémoire de ton adversaire au moment décisif.

### Audience cible
- **Cœur de cible** : 16-35 ans, joueurs casual mobile qui aiment la compétition rapide (parties < 3 min)
- **Secondaire** : Adultes 35+ qui jouent au memory en famille / pour entraîner leur cerveau
- **Niche locale** : Madagascar (fort potentiel, peu de concurrence locale sur les jeux de mémoire compétitifs)

### Différenciants à exploiter dans la fiche
1. **Tornade** — power-up unique qui efface la mémoire de l'adversaire (mécanique signature)
2. **Multijoueur en ligne réel** (la plupart des memory mobiles sont solo)
3. **Système ELO + classement** → addiction "ranked"
4. **3 niveaux d'IA** pour s'entraîner hors ligne
5. **Mode Casual avec ami** via code de room (partage rapide)
6. **Récompenses quotidiennes** + XP boost

## 📊 ÉTAPE 7 — Android Vitals (Performance)

### Cibles à maintenir pour Flipia

| Métrique | Seuil "bad" Google | Cible Flipia | Risque spécifique |
|---|---|---|---|
| Crash rate | > 1.09% | < 0.5% | Reanimated v4 sur low-end devices |
| ANR rate | > 0.47% | < 0.2% | Animations Tornade lourdes |
| Slow rendering | > 50ms | < 30ms | Grid 4×4 avec flip animations simultanées |
| Cold startup | > 5s | < 2.5s | Bundle size avec InstantDB + AdMob |

### Actions à mettre en place

1. **Sentry / Bugsnag** — monitoring crashes en production (à intégrer si pas déjà fait)
2. **Error Boundary** autour de `app/game/` et `app/room/` (gameplay = zone critique)
3. **EAS Update / OTA** — pour patcher rapidement les crashes sans passer par review
4. **Tester sur low-end** : émuler un appareil 2 GB RAM avec CPU throttling x4 dans Android Studio
5. **Vérifier le bundle size** : `npx expo-doctor` + analyser avec source-map-explorer

### Permissions
→ Auditer la liste des permissions dans `android/app/src/main/AndroidManifest.xml`. **Strict minimum** :
- ✅ INTERNET (multijoueur, AdMob)
- ✅ POST_NOTIFICATIONS (rappel daily reward)
- ❌ Retirer toute permission inutilisée — chaque permission inutile = +X% taux de désinstallation

## ⚠️ Pièges à éviter spécifiques à Flipia

1. **Confusion avec "Flippy" / "Flip" autres apps** → bien renforcer la marque "Flipia" dans la 1ère phrase de long desc
2. **Concurrence directe** : Match Pairs, Brain Training, etc. ont gros budget marketing → notre angle = "compétition multijoueur" pas "brain training"
3. **AdMob** : trop de pubs = uninstall rate ↑ = ranking ↓. Audit la fréquence (interstitials max 1 toutes les 3 parties)
4. **Onboarding trop long** = drop-off → mesurer le taux de complétion onboarding dans Play Console (Engagement)
5. **Mode hors ligne mal vendu** → c'est un VRAI atout commercial sur les marchés émergents (Madagascar, Afrique francophone). Le mettre en avant dans short desc.
6. **Catégorie "Cards" vs "Puzzle"** : tester les deux. Cards = moins concurrentiel mais audience plus orientée jeux d'argent. Puzzle = plus pertinent thématiquement mais ultra concurrentiel.
